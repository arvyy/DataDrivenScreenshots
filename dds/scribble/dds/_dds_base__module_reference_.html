1 (dds base) module reference.

The base module of dds. Module is included by default.

1.1 apply-data

Applies given data to item definition with given overrides. All properties expressed as functions of data get calculated, and returned value has all properties as values. Returns a copy - the argument is not modified.

(apply-data item-def data [ overrides ] )

  • item-def - item definition to apply data on.

  • data - data to apply.

  • optional overrides - list of overrides to use when applying data..( Default: empty list)


Return item definition with properties calculated

1.2 draw

Renders item definition either to screen or onto render texture; see (part "load-render-texture").

(draw item-def [ render texture ] )

  • item-def - item definition to be drawn; see item definition. All properties must have been resolved into exact values; see apply-data

  • optional render texture - render texture to drawn on, or #f to draw on screen..( Default: #f)


Return value unspecified

1.3 rect

Create rectangle item definition. Note: All key arguments, aside from id, are item definition properties. They either can be expressed as values, or as functions that take data as argument. Only item definitions that have values for all properties can be drawn; see apply-data for converting item with properties as function expressions into values

(rect
        #:id id
        #:x x
        #:y y
        #:width width
        #:height height
        #:roundness roundness
        #:fill fill
        #:stroke stroke
        #:stroke-width stroke-width
        #:segments segments)

  • #:id id - id; used in override. (Default: #f)

  • #:x x - left side x position. (Default: 0)

  • #:y y - top y position. (Default: 0)

  • #:width width - width. (Default: 100)

  • #:height height - height. (Default: 100)

  • #:roundness roundness - scale on how much rounding is done on corners, interval [0, 1]. (Default: 0)

  • #:fill fill - color to fill inside the shape, or #f to disable. (Default: (color 200 100 100))

  • #:stroke stroke - color to stroke shape, or #f to disable. (Default: (color 10 10 10))

  • #:stroke-width stroke-width - thickness of the stroke. (Default: 1)

  • #:segments segments - amount of segments used for rendering rounded corners. (Default: 20)


Returns rectangle item definition.

1.4 rect-o

Same as rect, except all defaults are #f instead. Meant to be used in overrides.

1.5 circle

Create circle item definition. Note: All key arguments, aside from id, are item definition properties. They either can be expressed as values, or as functions that take data as argument. Only item definitions that have values for all properties can be drawn; see apply-data for converting item with properties as function expressions into values

(circle
        #:id id
        #:x x
        #:y y
        #:radius radius
        #:inner-radius inner-radius
        #:start-angle start-angle
        #:end-angle end-angle
        #:fill fill
        #:stroke stroke
        #:stroke-width stroke-width
        #:segments segments)

  • #:id id - id; used in override. (Default: #f)

  • #:x x - center x position. (Default: 0)

  • #:y y - center y position. (Default: 0)

  • #:radius radius - outter radius of the circle. (Default: 100)

  • #:inner-radius inner-radius - inner radius of the circle. (Default: 0)

  • #:start-angle start-angle - sector start angle. (Default: 0)

  • #:end-angle end-angle - sector end angle. (Default: 0)

  • #:fill fill - color to fill inside the shape, or #f to disable. (Default: (color 200 100 100))

  • #:stroke stroke - color to stroke shape, or #f to disable. (Default: (color 10 10 10))

  • #:stroke-width stroke-width - thickness of the stroke. (Default: 1)

  • #:segments segments - number of segments used. (Default: 20)


Returns circle item definition

1.6 circle-o

Same as circle, except all defaults are #f instead. Meant to be used in overrides. See override.

1.7 triangle

Create triangle item definition. Note: All key arguments, aside from id, are item definition properties. They either can be expressed as values, or as functions that take data as argument. Only item definitions that have values for all properties can be drawn; see apply-data for converting item with properties as function expressions into values

(triangle
        #:id id
        #:points points
        #:fill fill

        #:stroke-width stroke-width)

  • #:stroke stroke - color to stroke shape, or #f to disable. (Default: (color 10 10 10))

  • #:stroke-width stroke-width - thickness of the stroke. (Default: 1)


Returns triangle item definition.

1.11 t-rect

Draw a texture. x,y,width,height define rectangle in draw target dimension onto which the texture will be drawn. src-x, src-y, src-width, src-height define rectangle in texture dimension, which texture region will be drawn.

(t-rect
        #:id id
        #:x x
        #:y y
        #:width width
        #:height height
        #:src-x src-x
        #:src-y src-y
        #:src-width src-width
        #:src-height src-height
        #:color color
        #:texture texture)

  • #:id id - id; used in override. (Default: #f)

  • #:x x - left side x position. (Default: 0)

  • #:y y - top y position. (Default: 0)

  • #:width width - width. Can be #f, in which case texture width is used. (Default: #f)

  • #:height height - height. Can be #f, in which case texture height is used. (Default: #f)

  • #:src-x src-x - left side x position in texture. (Default: 0)

  • #:src-y src-y - top y position in texture. (Default: 0)

  • #:src-width src-width - width of texture region. Can be #f, in which case texture height is used. (Default: #f)

  • #:src-height src-height - height texture region. Can be #f, in which case texture height is used. (Default: #f)

  • #:color color - color to tint the texture. (Default: (color 255 255 255))

  • #:texture texture - texture to be drawn. (Default: #f)


Returns texture rectangle definition.

1.12 t-rect-o

Same as t-rect, except all defaults are #f instead. Meant to be used in overrides.

1.13 text

Draws text. See load-font.

(text
        #:id id
        #:text text
        #:x x
        #:y y
        #:font font
        #:font-size font-size
        #:spacing spacing
        #:color color)

  • #:id id - id; used in override. (Default: #f)

  • #:text text - The text to be drawn. (Default: "")

  • #:x x - x position. (Default: 0)

  • #:y y - y position. (Default: 0)

  • #:font font - font to use, or #f for default. (Default: #f)

  • #:font-size font-size - font size. (Default: 12)

  • #:spacing spacing - space between letters in text. (Default: 2)

  • #:color color - text color. (Default: (color 10 10 10))


Returns text item definition

1.14 text-o

Same as text, except all defaults are #f instead. Meant to be used in overrides.

1.15 cnt

Container for grouping items, applying transformations, and shaders.

(cnt
        #:id id
        #:transform transform
        #:blend blend
        #:post-processing post-processing
        #:items items)

  • #:id id - id; used in override. (Default: #f)

  • #:transform transform - 3*2 transformation matrix of the container. See combine, rotate, translate, scale. (Default: (translate 0 0))

  • #:blend blend - blending used for items, expressed as a pair of source and destination factors represented as symbols. Symbols are same as OpenGL constants; eg. (cons ’GL_ONE ’GL_SRC_ALPHA). Can be #f to disable blending. See blending. (Default: #f)

  • #:post-processing post-processing - list of shaders to use. Shader can be shader, or can be a function of data that returns a shader. See shader, shaders.. (Default: #f)

  • #:items items - Not a property. Either a list of child item definitions, or a function for data propagation. Function takes data as argument, and returns list of pairs - each pair is of child item def and child datum. See cnt-items-tpl, cnt-items-transfempty list. (Default: empty list)


Returns container item definition

1.16 cnt-o

Same as cnt-o, except all defaults are #f instead. Meant to be used in overrides. Note, that "items" is not overrideable.

1.17 color

Create RGB color structure.

(color r g b [ a ] )

  • r - red; 0-255

  • g - green; 0-255

  • b - blue; 0-255

  • optional a - alpha; 0-255.( Default: 255)


Returns a color structure.

1.18 combine

Combine multiple transformation matrices into one.

(combine transform ... )

  • transform ... - One or more 3*2 transformation matrices to combine. See translate, scale, rotate


Returns new transformation matrix

1.19 rotate

Create rotation transformation matrix.

(rotate angle)

  • angle - angle in degrees by which rotate around origin clock wise


Returns transformation matrix

1.20 translate

Create translation transformation matrix.

(translate x y)

  • x - translation along x axis

  • y - translation along y axis


Returns tranformation matrix

1.21 scale

Create scaling transformation matrix.

(scale xk yk)

  • xk - scale factor along x axis

  • yk - scale factor along y axis


Returns transformation matrix.

1.22 calc

calc is same as a lambda with one argument, except instead of a simple variable binding, a pattern match is used.

ie (calc (pattern) ...) is equivalent to (lambda (arg) (match arg (pattern ...)))

See Guile pattern matching href.

1.23 cnt-items-tpl

Allows creating multiple container items with same definition, but different data. See cnt.

(cnt-items-tpl item-def children-data)

  • item-def - item definition used as template

  • children-data - either a list of data, or a function of data that returns list of data


Returns function conformant with cnt #:items

1.24 cnt-items-transf

Allows converting data into another data. Might be useful for using a common subtree which has defined data format.

(cnt-items-transf item-def-list data-transformer)

  • item-def-list - list of items

  • data-transformer - function that takes data, and returns transformed data


Returns function conformant with cnt items

1.25 compute

Syntax for deriving property from existing properties.

(compute (arguments ...) computation-body) . Arguments can be either values, or functions of data that returns value. Syntax produces a function of data, which when applied executes the computation body, which is suitable to be used as a property in item definitions.

1.26 shader

Prepares to use a shader inside container. Each uniform must match one of the following patterns. See get-shader-loc for obtaining location value.

  • (’vec location value) - Set vector uniform. Must be a scheme vector, eg (vector 1 2). Floats are also set this way using a vector of length 1.

  • (’cnt-transform location) - Set matrix uniform with the value of current container transformation. Useful when shader needs to be applied not in screen dimension, but in container.

  • (’matrix location value) - Set matrix uniform.

  • (’matrix/invert location value) - Set matrix uniform, which gets inverted before being set.

  • (’texture location value) - Set texture (sampler2d) uniform. Note, that max 3 textures may be used in same shader. See load-texture, render-texture->texture



(shader shader-id uniforms ... )

  • shader-id - shader to use. See load-shader

  • uniforms ... - zero or more uniforms for given shader.


Returns a shader for use in cnt #:post-processing

1.27 override

"Override any item matching the path and type with provided properties. In some way similar to CSS. "

(override cnt-branch-pred override-item-def)

  • cnt-branch-pred - predicate function for testing if this override matches a given container branch. Branch is expressed as a list of id of containers in the branch; last item in branch might another item. See pattern

  • override-item-def - override item definition. Each item property must be either a value, or a function of initial (before override) value and datum for the item


Returns an override definition to be used in apply-data

1.28 pattern

Syntax for defining predicate function that returns true if pattern matches input. Used with override.

(pattern pattern ...) is equivalent to (lambda(branch) (match branch (pattern #t) (_ #f)))

1.29 dynamic-item

Create dynamic depending on provided data. Note, that returned item gets data applied to it at later step.

(dynamic-item item-fact)

  • item-fact - function that takes data as argument, and returns item definition


Returns dynamic item definition

1.30 load-texture

Load texture from file.

(load-texture file)

  • file - path to file to load as a texture


Texture reference

1.31 create-render-texture

Create a render texture, onto which you can draw stuff. See draw.

(create-render-texture width height)

  • width - width of texture in pixels

  • height - height of texture in pixels


Returns render texture reference

1.32 clear-render-texture

Clears render texture with blank color.

(clear-render-texture renderTextre)

  • renderTextre - reference to render texture. See load-render-texture


Return is unspecified.

1.33 render-texture->texture

Prepare render texture to be used as texture. Note that this function doesn’t copy - changes to render texture will also mean changes to this returned texture.

(render-texture->texture renderTexture)

  • renderTexture - render texture reference. See create-render-texture


Returns reference to texture

1.34 load-shader

Load shader from file.

(load-shader vertexFile fragmentFile)

  • vertexFile - vertex shader file path, or #f to use a default vertex shader

  • fragmentFile - fragment shader file path, or #f to use a default fragment shader


Returns shader reference

1.35 get-shader-loc

Get shader location using uniform name.

(get-shader-loc shader uniformName)

  • shader - shader reference. See load-shader

  • uniformName - uniform name in the shader


Returns shader location id

1.36 load-font



(load-font font-file)

  • font-file - font file path


Returns font reference
